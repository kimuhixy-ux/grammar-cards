## Phase 2: 文法項目統合

- [x] `grammar_items.json` 生成（`pipeline/02_merge_items.py`）
- [x] 文法カテゴリ一覧を出力して目視チェック（`pipeline/output/grammar_categories.txt`）
- [x] 総項目数: 719件（採用したページ内項目751件のうち、`grammar_category`完全一致で19カテゴリが複数ソースに統合）
- 設計判断: カテゴリの統合は`grammar_category`文字列の完全一致のみで行い、意味的に近い表記ゆれ
  （例: 「前置詞」「関係代名詞」「受動態」等の細分化されたバリエーション）はあえて統合しなかった。
  各項目が特定ページの具体的な文法ポイントに対応するため、Phase 3のカード生成で項目ごとに
  3〜5問生成する設計と相性が良く、粒度を保つ方がカードバンクの多様性につながると判断。
  （もし後で「前置詞」等を大分類でも束ねたくなった場合は、`grammar_category`にLLMベースの
  正規化ステップを挟む改修が必要。現時点では未実施・保留）
- ページ番号の扱いに関する既知の注意点: 抽出JSON内部の`page`フィールドは、モデルが印刷ページ番号を
  読み取って返す場合があり、キャッシュファイル名の連番（PDFページ順）と一致しないことがある
  （例: chigasaki PDFページ100 → 内部page値199、chigasaki PDFページ158 → 内部page値314）。
  `grammar_items.json`の`sources[].page`は常にキャッシュファイル名（PDFページ順）由来の値を採用しており、
  信頼できる。
