# はじめかた.md — GrammarCards

## 1. プロジェクトフォルダを作る

```bash
mkdir -p ~/grammar_cards/source_pdfs
cd ~/grammar_cards
```

`CLAUDE.md`・`進捗.md`・`はじめかた.md`（このファイル）をこのフォルダ直下に置く。

## 2. 元PDFを配置する

```bash
cp /path/to/総合英語forest.pdf ~/grammar_cards/source_pdfs/forest.pdf
cp /path/to/茅ヶ崎方式国際英語教本文法.pdf ~/grammar_cards/source_pdfs/chigasaki.pdf
```

`.gitignore` に以下を必ず入れる（著作権物のためリポジトリにコミットしない）:

```
source_pdfs/
pipeline/cache/
pipeline/pages/
```

## 3. Claude Codeを起動して最初に貼るプロンプト

```
cd ~/grammar_cards
claude
```

起動後、以下をそのまま貼り付ける:

```
このプロジェクトの CLAUDE.md を読み込んで、書いてある通りにPhase 0（セットアップとPoC）から
着手してください。作業を始める前に、まず進捗.mdを確認し、作業が終わるたびに進捗.mdのチェック
ボックスを更新してください。

最初のゴールはPhase 0です。source_pdfs/forest.pdf と source_pdfs/chigasaki.pdf から
それぞれ10ページ程度をサンプルとして画像化し、Vision APIで構造化抽出するPoCスクリプトを
作成・実行し、抽出結果を私がレビューできる形で見せてください。全ページ処理はまだ行わないで
ください。
```

PoCの結果を見て、抽出精度に問題があればプロンプトを直させてから次のPhaseに進む。
「Phase 1に進めてください」「進捗.mdの続きから再開してください」だけで以降のセッションは十分。

## 4. 全ページ処理を始める前の確認事項

- APIコストの見積もりをClaude Codeに出させてから実行を許可すること（800ページ分の画像入力は
  それなりのコストになる）
- Batches APIはジョブ完了まで時間がかかる（最大24時間程度）ので、投入後は放置してよい設計に
  なっているか確認する
- 途中でエラーが出ても、キャッシュ済みページは再実行時にスキップされる設計になっているか確認する

## 5. 完成後の日々の使い方

- iPhoneのSafariでGitHub PagesのURLを開き、共有ボタン→「ホーム画面に追加」
- 以後はアプリのように起動でき、オフラインでも学習状態が端末内に保存される
- 教材を追加したくなった場合は、新しいPDFを `source_pdfs/` に置いてPhase 1〜3のスクリプトを
  再実行すれば、既存のカードバンクに追加される設計にしておくとよい
